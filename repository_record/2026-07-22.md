## 核心决策（均经与语文教师确认）
- **题型默认混合**：未指定题型时默认「混合」（一半上下句填空 + 一半理解性默写），系统不询问题型倾向；仅当教师明确要求单一题型时按其指定出题。此前（2026-07-21）默认为上下句填空。
- **篇目严格限定教材范围**：出题篇目严格限定在教材知识库范围内，不含任何课外古诗词，系统也不提供「加入课外篇目」选项。
- **篇目自动全覆盖**：给定年级+学期后系统自动在该范围内全部篇目中抽取出题，教师无需也无法逐篇挑选，避免增加决策负担。

### 3. 三个 Tool（`src/backend/app/services/tools/`）
- [x] `query_poems`：按年级+学期检索篇目，返回摘要（不含全文，避免 token 膨胀），给出 `content_length` 供多样性策略参考。
- [x] `select_questions`：核心出题逻辑——
  - 按标点（句号/问号/叹号/分号）拆句，过滤过短/过长/对话密度过高的句子。
  - 识别上下句对偶（逗号分隔的两分句）。
  - 按题型选题：上下句填空优先选有对偶结构的句子；理解性默写答案句选中后题干由 LLM 生成；混合题型各占一半，奇数向下取整给理解性默写。
  - 篇目多样性：第一轮每篇最多 1 题，不足时放宽。
  - 上下句填空随机决定「给上句填下句」或「给下句填上句」。
- [x] `get_error_prone_chars`：取篇目级易错字，无则返回空数组（按需标注）。

### 4. Skill 与 Agent 编排（`src/backend/app/services/`）
- [x] `skills/output_format.py`：格式化输出，题目与【答案与解析】分段，题型统计描述，易错字按需标注（无则不输出该行）。
- [x] `quiz_agent.py`：Agent 主流程，采用「后端代码编排」方式（MVP 方案）——
  1. LLM 解析教师自然语言需求 → `{grade, semester, count, question_type}`，支持从对话历史继承缺失信息（多轮追问）。
  2. 范围不完整时追问（缺年级学期问哪个年级学期；缺学期问上册下册）。
  3. `query_poems` 检索篇目。
  4. `select_questions` 选题。
  5. LLM 为理解性默写题生成语义题干（题干不得包含答案原句）。
  6. `get_error_prone_chars` 逐题取易错字。
  7. `format_quiz_output` 格式化输出。
  - 知识库单例懒加载；JSON 提取兼容 markdown 代码块与噪声。
