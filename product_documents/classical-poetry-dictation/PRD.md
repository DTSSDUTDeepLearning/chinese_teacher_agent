# 古诗文默写出题系统 - 产品需求文档

## 1. 项目概述

### 1.1 项目背景
初中语文教师在日常教学中需要频繁为学生准备古诗文默写练习题。目前教师需要手动从教材和考试提纲中挑选篇目、设计题目，耗时耗力。本项目旨在通过AI智能体（Agent）技术，为教师提供一个自动化的古诗文默写出题工具，提升备课效率。

### 1.2 项目目标
- 为初中语文教师提供智能古诗文默写出题服务
- 题目来源于浙江省中考考试提纲，确保符合考试要求
- 默认生成6道题（与中考题型一致），默认题型为混合（上下句填空 + 理解性默写），教师可按需指定其他数量或单一题型
- 支持上下句填空和理解性默写两种题型，以及二者混合
- 在对话框中直接输出格式化的题目和解析，教师可自行复制使用

### 1.3 目标用户
**主要用户**：初中语文教师
- 需要为学生准备课堂练习、课后作业或考试试卷
- 希望快速生成符合中考标准的默写题目
- 不熟悉复杂技术操作，偏好自然语言交互

## 2. 功能需求

### 2.1 核心功能

| 功能 | 描述 |
|------|------|
| 智能出题 | 教师通过自然语言描述需求，系统自动生成6道古诗文默写题（默认混合题型） |
| 范围筛选 | 支持按年级、学期筛选出题范围（如"七年级"、"八年级下学期"、"整个三年"）；给定范围后自动覆盖该范围内全部篇目，无需教师逐篇挑选，且不含任何课外古诗词 |
| 对话输出 | 在对话框中直接输出格式化的题目和解析，教师可自行复制使用 |
| 对话交互 | 通过对话框与Agent交互，支持追问和补充条件 |

### 2.2 功能详情

#### 2.2.1 出题规则
- **题型**：支持两种题型，默认混合
  - **上下句填空**：如"______，后天下之乐而乐"
  - **理解性默写**：给出语义描述题干，要求学生写出对应古诗文句子，如"《岳阳楼记》中表达旷达胸襟的句子是______"
  - **混合（默认）**：一半上下句填空 + 一半理解性默写，打乱顺序输出；教师无需指定题型，系统也不会询问题型倾向。仅当教师明确要求单一题型时，按其指定出题
- **数量**：默认6道（与中考保持一致），教师可指定其他数量（如5/10/20道）
- **来源**：浙江省中考考试提纲中的古诗文篇目，严格限定在教材知识库范围内，**不包含任何课外古诗词**，系统也不提供"加入课外篇目"的选项
- **篇目选择**：给定年级+学期后，系统自动在该范围内的全部篇目中抽取出题，教师无需也无法逐篇挑选，避免增加决策负担
- **易错字标注**：仅当题目答案中存在易错字、易混淆字、生僻字时才标注，无则不标
  - 如"学而不思则罔，思而不学则殆"中的"罔"（易错为"惘"）和"殆"（易错为"怠"）
  - 在答案解析中提供易错字说明
  - 若该题无易错字，则不输出易错字行，避免噪音

#### 2.2.2 筛选条件
教师可通过自然语言指定以下筛选维度：

| 维度 | 可选值 | 示例 |
|------|--------|------|
| 年级 | 七年级 / 八年级 / 九年级 / 全部 | "出八年级的题" |
| 学期 | 上册 / 下册 / 全部 | "八年级下学期" |
| 范围 | 指定年级学期 / 整个初中 | "整个三年" |
| 题型 | 混合（默认） / 上下句填空 / 理解性默写 | 默认混合；指定单一题型可说"出理解性默写" |
| 数量 | 默认6道 / 教师指定 | "出10道题" |

> **经与语文教师确认（2026-07-21）**：本期筛选维度仅保留「年级 + 学期」两个篇目筛选维度，不再引入「体裁」「重要性/频率」作为教师可筛选维度。篇目级考试频率仍作为内部出题加权使用，但不暴露为筛选条件。新增「题型」「数量」两个出题参数，供教师选择题型与题量。

> **经与语文教师确认（2026-07-22）**：出题篇目严格限定在教材知识库范围内，不含任何课外古诗词，系统也不提供该选项；给定年级+学期后自动覆盖全部篇目，教师无需逐篇挑选；题型默认混合，系统不询问题型倾向。

**默认行为**：
- 若教师未指定任何筛选条件，Agent主动询问年级和学期
- 若教师说"整个三年"，则从七年级到九年级全部篇目中出题
- 若教师未指定题型，默认为混合题型（一半上下句填空 + 一半理解性默写）
- 若教师未指定数量，默认为6道

#### 2.2.3 交互方式
- 教师直接在对话框中输入自然语言需求
- Agent解析需求后生成题目，直接在对话框中输出
- 若需求信息不完整，Agent可追问补充

**示例对话**：
```
教师：帮我出6道古诗文默写题，八年级下学期的
Agent：好的，为您生成八年级下学期的古诗文默写题：

1. ______，在河之洲。（《诗经·关雎》）
2. 蒹葭苍苍，______。（《诗经·蒹葭》）
...

【答案与解析】
1. 关关雎鸠
   易错字："雎"不要写成"睢"
2. 白露为霜
   易错字："葭"不要写成"暇"
```

#### 2.2.4 输出格式
- **输出方式**：在对话框中直接输出格式化的文本
- **输出结构**：
  - **题目部分**：6道填空题，每题一行，包含出处提示
  - **答案与解析部分**：紧跟题目后，包含每题的答案和易错字说明
- **格式要求**：
  - 使用清晰的编号和换行
  - 易错字用引号标注，提供简洁的辨析说明（是否是易错字，由AI自动判别）
  - 若某题无易错字，则不输出易错字行（按需标注，避免噪音）
  - 便于教师直接复制到Word或其他文档中使用

## 3. 用户场景

### 3.1 典型使用场景

**场景1：课堂练习出题**
- 教师准备明天的语文课，需要一份古诗文默写练习
- 打开系统，输入"帮我出6道七年级上册的古诗文默写题"
- 系统在对话框中输出题目和解析，教师复制后粘贴到Word中打印分发

**场景2：期中复习出题**
- 期中考试临近，教师需要综合复习题
- 输入"出6道八年级的古诗文默写题，上下册都要"
- 系统从八年级上下册全部篇目中根据考试频率加权抽取生成题目（默认混合题型）

**场景3：中考模拟训练**
- 初三教师准备中考模拟卷
- 输入"帮我出6道古诗文默写题，整个初中三年的范围"
- 系统从七年级到九年级全部篇目中出题

**场景4：未指定范围的交互**
- 教师输入"帮我出6道古诗文默写题"
- Agent追问："请问您需要哪个年级、哪个学期的题目？"
- 教师补充："八年级下学期"
- Agent生成对应范围的题目

## 4. 数据需求

### 4.1 题库数据
**核心数据源**：浙江省中考考试提纲中的古诗文篇目

**数据内容**：
- 篇目名称（如《岳阳楼记》《登高》）
- 作者信息
- 年级学期归属（七年级上/下、八年级上/下、九年级上/下）
- 全文内容（标准文本）
- **易错字标注**（关键）：
  - 每句诗中的易错字、易混淆字、生僻字
  - 如"学而不思则罔"中的"罔"（易错为"惘"）
  - 如"思而不学则殆"中的"殆"（易错为"怠"）

**数据获取方式**：
- 需要整理浙江省中考考试提纲中的古诗文篇目清单
- 将数据放入知识库，让AI限定从该知识库中获取题目，并根据频率加权出题

### 4.2 用户数据
**会话历史**：
- 同一会话内保存对话记录，使用Agent自带的记忆机制
- 无需跨会话保存历史记录
- 无需用户账号体系（本地运行）

---

## 5. 非功能需求

### 5.1 性能要求
- 出题响应时间：10秒内完成6道题目的生成
- 知识库检索：快速匹配篇目范围，无明显延迟

### 5.2 可用性要求
- 对话式交互，零学习成本
- 输出格式清晰，便于教师直接复制到文档中使用
- 若需求信息不完整，Agent主动引导补充

### 5.3 准确性要求
- 题目内容必须与原文完全一致，不允许出现错别字或漏字
- 出处标注准确（篇名、作者）
- 题目不得超出教师指定的范围
- 易错字、易混淆字标注必须准确，不得误导学生

---

## 6. 约束条件

### 6.1 技术约束
- **运行方式**：本地运行，暂不考虑Web部署、桌面应用或小程序
- **AI模型**：通过智能Agent调用大语言模型生成题目
- **知识库**：题库数据需放入知识库，限定AI从知识库范围内出题
- **前端**：复用现有的Vue对话式UI（`src/frontend`）

### 6.2 业务约束
- 题目数量默认为6道，与浙江省中考题型保持一致；教师可指定其他数量
- 题型支持上下句填空、理解性默写，以及二者混合；未指定时默认混合
- 题库范围严格限定为浙江省中考考试提纲，不含任何课外古诗词；给定年级+学期后自动覆盖全部篇目，教师无需逐篇挑选
- 输出格式为对话框文本，便于教师复制使用
