# 古诗文默写出题系统 - 技术设计文档

## 1. 需要教师提供的输入材料

在开发之前，需要教师提供以下材料，作为知识库构建和系统设计的依据：

### 1.1 必需材料

| 材料 | 用途 | 说明 |
|------|------|------|
| **浙江省中考考试提纲** | 确定知识库范围 | 决定哪些古诗文篇目需要纳入知识库，这是最核心的输入 |
| **题型样例** | 对齐出题格式 | 提供真实中考古诗文默写题的样例（含上下句填空与理解性默写），确保系统输出与中考一致 |

### 1.2 经验性材料（由教师整理提供，可选）

> **说明**：根据 PRD 2.2.2 与教师确认（2026-07-21），「体裁」「重要性/频率」**不再作为教师可筛选维度**。下列材料仅用于**内部出题加权**，不暴露为筛选条件。无这些材料时系统仍可正常出题（退化为均匀随机）。

| 材料 | 用途 | 说明 |
|------|------|------|
| **易错字/易混淆字清单** | 构建易错字标注数据 | 教师根据教学经验，标注每篇古诗文中学生容易写错的字；缺失部分由 LLM 兜底生成 |
| **中考真题/模拟卷** | 统计考试频率（内部加权） | 用于分析篇目级别和诗句级别的出题频率，**仅用于内部加权出题，不作为筛选条件** |
| **篇目重要性分级** | 辅助频率加权（内部） | 教师对篇目重要性的主观判断（S/A/B级），仅内部使用 |
| **常考名句清单** | 诗句级频率参考（内部） | 教师经验中"必考"的句子 vs 低频句子，仅内部使用 |

### 1.3 材料格式建议

**古诗文篇目清单**（建议用表格或JSON，体裁为可选内部字段）：
```
篇目名称 | 作者 | 年级 | 学期 | 体裁(可选) | 全文
岳阳楼记 | 范仲淹 | 九年级 | 下册 | 文 | 庆历四年春...
```

**考试频率数据**（可选，仅用于内部加权，不暴露为筛选条件）：
```
篇目名称 | 篇目频率(近5年考次) | 高频诗句 | 诗句频率
岳阳楼记 | 8次 | 不以物喜，不以己悲 | 5次
岳阳楼记 | 8次 | 先天下之忧而忧，后天下之乐而乐 | 4次
```

**易错字数据**（建议用表格）：
```
篇目 | 诗句 | 易错字 | 常见错误写法 | 辨析说明
论语 | 学而不思则罔 | 罔 | 惘 | "罔"意为迷惑，"惘"意为失意
论语 | 思而不学则殆 | 殆 | 怠 | "殆"意为有害，"怠"意为懈怠
```

## 2. 技术选型

### 2.1 知识库方案：结构化数据（非RAG）

**结论：不需要RAG，使用结构化数据（JSON/SQLite）即可。**

**理由**：
- 古诗文数据是**高度结构化**的：篇目名称、作者、年级学期、全文、频率、易错字等都是确定性字段
- 不需要"语义检索"——教师指定"八年级下册"就是精确匹配，不存在模糊语义
- 数据量可控（浙江省中考提纲约60-80篇古诗文），不需要向量索引
- 频率加权、易错字标注都需要精确的结构化字段，RAG反而增加复杂度

**知识库数据结构设计**：

> 说明：`genre`、`importance`、`exam_frequency` 等字段**仅用于内部出题加权**，不作为教师可筛选维度暴露（见 PRD 2.2.2）。理解性默写的语义题干不预存，由 LLM 基于诗句内容即时生成，以保证题干多样性和教学合理性。

```json
{
  "poems": [
    {
      "id": "poem_001",
      "title": "岳阳楼记",
      "author": "范仲淹",
      "grade": "九年级",
      "semester": "下册",
      "genre": "文",              // [内部字段] 仅用于内部统计，不暴露为筛选条件
      "importance": "S",          // [内部字段] 仅用于内部加权
      "exam_frequency": {         // [内部字段] 仅用于内部加权；缺失时退化为均匀随机
        "poem_level": 8,
        "line_level": {
          "不以物喜，不以己悲": 5,
          "先天下之忧而忧，后天下之乐而乐": 4,
          "庆历四年春": 1
        }
      },
      "content": "庆历四年春，滕子京谪守巴陵郡...",
      "lines": [
        {
          "text": "不以物喜，不以己悲",
          "prev_line": "居庙堂之高则忧其民，处江湖之远则忧其君",
          "next_line": "是进亦忧，退亦忧",
          "error_prone_chars": [
            {
              "char": "己",
              "confused_with": "已",
              "hint": "\"己\"是自己，\"已\"是已经"
            }
          ],
          "frequency": 5,         // [内部字段] 仅用于内部加权
          "comprehension_hints": ["表达旷达胸襟", "不因外物好坏和自己得失而或喜或悲"]  // 可选：辅助 LLM 生成理解性默写题干
        }
      ]
    }
  ]
}
```

**存储方案**：
- **开发阶段**：JSON文件（`src/backend/data/poems.json`），便于快速迭代
- **稳定阶段**：SQLite（`src/backend/data/poems.db`），便于查询和索引
- 通过Python模块封装数据访问层，上层代码不感知存储变化

## 3. Tools 设计

Agent通过调用以下Tools完成出题任务：

### 3.1 核心Tools

| Tool名称 | 功能 | 输入 | 输出 |
|----------|------|------|------|
| `query_poems` | 查询知识库中的古诗文篇目 | grade, semester | 符合条件的篇目列表 |
| `select_questions` | 根据频率加权选出题目 | poem_ids, count=6, question_type="context_fill" | 选中的题目（含诗句、出处、频率、题型） |
| `get_error_prone_chars` | 获取指定诗句的易错字信息 | poem_title, line_text | 易错字列表及辨析说明（无则返回空数组） |

> `question_type` 取值：`context_fill`（上下句填空，默认）、`comprehension`（理解性默写）、`mixed`（混合）。详见 PRD 2.2.1。

### 3.2 各Tool详细说明

#### `query_poems`
```
输入：{
  "grade": "八年级",        // 可选，不传则查全部
  "semester": "下册"        // 可选
}
输出：[
  { "id": "...", "title": "...", "author": "...", "grade": "...", "semester": "...", "importance": "S", "exam_frequency": 8 },
  ...
]
```
- 纯数据查询，不涉及LLM
- 从知识库中按条件过滤
- **不接收 `genre`（体裁）参数**：根据 PRD 2.2.2，体裁不再作为教师可筛选维度；如内部需要可由 KnowledgeBase 直接读取

#### `select_questions`
```
输入：{
  "poem_ids": ["poem_001", "poem_002", ...],   // 候选篇目
  "count": 6,                                   // 默认6，教师可指定其他数量（如5/10/20）
  "question_type": "context_fill"               // "context_fill"(默认) / "comprehension" / "mixed"
}
输出：[
  {
    "poem_title": "岳阳楼记",
    "question_type": "context_fill",            // 本题题型
    "given_line": "不以物喜，______",           // 上下句填空时的题面
    "answer_line": "不以己悲",
    "frequency": 5,                             // [内部] 加权用
    "direction": "上句填下句"                   // 或 "下句填上句"
  },
  {
    "poem_title": "岳阳楼记",
    "question_type": "comprehension",           // 理解性默写
    "given_line": "《岳阳楼记》中表达旷达胸襟的句子是______",  // 由 LLM 生成语义题干
    "answer_line": "不以物喜，不以己悲",
    "frequency": 5
  },
  ...
]
```
- 核心出题逻辑：根据频率加权随机抽取（频率数据缺失时退化为均匀随机）
- 频率越高的诗句被选中概率越大
- 尽量保证题目来自不同篇目（多样性）；题目数 > 候选篇目数时再放宽
- 上下句填空：随机决定"给上句填下句"还是"给下句填上句"
- 理解性默写：由 LLM 基于 `comprehension_hints`（若有）或诗句语义即时生成题干，题干不得直接包含答案原句
- `mixed`：按教师意图分配两种题型数量（如"出一半理解性默写"则各占一半，遇奇数向下取整给理解性默写）

#### `get_error_prone_chars`
```
输入：{
  "poem_title": "论语",
  "line_text": "学而不思则罔，思而不学则殆"
}
输出：[
  { "char": "罔", "confused_with": "惘", "hint": "\"罔\"意为迷惑，\"惘\"意为惆怅" },
  { "char": "殆", "confused_with": "怠", "hint": "\"殆\"意为有害，\"怠\"意为懈怠" }
]
```
- 从知识库中查询预标注的易错字数据
- 若知识库无标注，可回退到LLM生成（但标注优先）
- **按需标注**：仅当存在易错字、易混淆字、生僻字时才返回；无易错字时返回**空数组 `[]`**，上层据此**不输出易错字行**，避免噪音（见 PRD 2.2.1）
- 是否为易错字由 AI 自动判别，不过度标注

### 3.3 不需要的Tools

| 不需要的Tool | 原因 |
|-------------|------|
| 读古诗文全文的Tool | 全文已存在知识库中，`query_poems`返回时已包含 |
| 区分高频/低频的独立Tool | 频率数据已结构化存储在知识库中，`select_questions`直接使用 |
| RAG检索Tool | 数据结构化程度高，不需要语义检索 |

## 4. Skills 设计

Skills是可复用的经验总结，封装为Agent可调用的能力模块。

### 4.1 建议的Skills

| Skill名称 | 描述 | 内容 |
|-----------|------|------|
| **出题策略Skill** | 如何科学地选出题目 | 题型分配（上下句/理解性/混合）、频率加权算法、篇目多样性策略、上下句方向随机策略 |
| **易错字分析Skill** | 如何标注和解析易错字 | 从知识库读取预标注数据，格式化为教学友好的解析文本；按需标注，无则不输出 |
| **对话输出格式化Skill** | 如何在对话框中输出清晰的题目和解析 | 题目编号格式、答案与解析的组织方式、便于教师复制的排版 |

### 4.2 Skill详细说明

#### 出题策略Skill
```
核心经验规则：
1. 题型分配：
   - context_fill（上下句填空，默认）：给上句填下句或给下句填上句，方向随机
   - comprehension（理解性默写）：由 LLM 生成语义题干，题干不得包含答案原句
   - mixed（混合）：按教师意图分配两种题型数量（如"出一半理解性默写"各占一半，奇数向下取整给理解性默写）
2. 频率加权：高频诗句被选中概率 = 该诗句频率 / 所有候选诗句频率之和（频率缺失时退化为均匀随机）
3. 篇目多样性：题目尽量来自不同篇目（若候选篇目数 ≥ count，则每篇最多1题）
4. 难度均衡：高频篇目多出题，但也保留1-2道低频题作为补充
```

#### 易错字分析Skill
```
核心经验规则：
1. 优先使用知识库中的预标注数据（准确性高）
2. 若知识库无标注，由LLM根据教学经验补充生成
3. 解析格式：字义解释 + 常见错误 + 区分方法
4. 按需标注：只标注真正容易写错的字，不过度标注；无易错字则不输出易错字行（避免噪音）
```

#### 对话输出格式化Skill
```
核心经验规则：
1. 题目部分：使用数字编号，每题一行，包含出处提示
   - 上下句填空：给出带空格的诗句，如"______，在河之洲。（《诗经·关雎》）"
   - 理解性默写：给出语义题干 + 空格，如"《岳阳楼记》中表达旷达胸襟的句子是______"
2. 答案与解析部分：紧跟题目后，使用【答案与解析】标题分隔
3. 易错字提示：使用引号标注易错字，提供简洁的辨析说明；按需标注，无易错字则不输出该行
4. 整体格式：便于教师直接复制到Word或其他文档中使用
```

## 5. 非Agent类工程实现

### 5.1 对话输出格式化

**技术选型**：纯字符串拼接，无需额外库

**输出模板**：
```
好的，为您生成八年级下学期的古诗文默写题（共6道，题型：3上下句填空 + 3理解性默写）：

1. ______，在河之洲。（《诗经·关雎》）
2. 蒹葭苍苍，______。（《诗经·蒹葭》）
3. 《岳阳楼记》中表达旷达胸襟的句子是______。（范仲淹）
4. 《桃花源记》中描写老人小孩都怡然自得的句子是______。（陶渊明）
5. ______，志在千里。（《龟虽寿》曹操）
6. 《论语》中阐述"学"与"思"关系的句子是______。

【答案与解析】
1. 关关雎鸠
   易错字："雎"不要写成"睢"（睢县）

2. 白露为霜
   易错字："葭"不要写成"暇"（闲暇）

3. 不以物喜，不以己悲
   易错字："己"不要写成"已"（已经）

4. 黄发垂髫，并怡然自乐
   易错字："髫"不要写成"髪"（髪指头发）

5. 老骥伏枥
   （无易错字，不输出易错字行）

6. 学而不思则罔，思而不学则殆
   易错字："罔"不要写成"惘"（惘意为惆怅），"殆"不要写成"怠"（怠意为懈怠）
```

**实现方式**：
- Agent调用Tools获取题目和易错字数据后，直接拼接成上述格式
- 通过Chat API的`content`字段返回给前端
- 前端使用Markdown渲染（已有`marked`库支持）
- **按需标注易错字**：`get_error_prone_chars` 返回空数组时，该题答案下不输出"易错字"行
- **题型混排**：`question_type=mixed` 时，上下句填空与理解性默写按分配数量混排编号
- **数量可变**：`count` 由教师指定时，编号与题目数随之变化

### 5.3 Agent与Tools的集成

**方案**：使用DeepSeek的Function Calling能力

```python
# Agent核心流程伪代码
tools = [
    query_poems_tool,
    select_questions_tool,
    get_error_prone_chars_tool,
]

system_prompt = """你是一位初中语文古诗文出题助手。
当教师要求出题时，按以下步骤操作：
1. 解析教师的年级、学期范围要求；若缺失则主动追问
2. 解析题型要求（上下句填空/理解性默写/混合），未指定则默认上下句填空
3. 解析数量要求，未指定则默认6道
4. 调用 query_poems 查询符合条件的篇目
5. 调用 select_questions 根据频率加权选出题目（传入 count 与 question_type）
6. 对每道题调用 get_error_prone_chars 获取易错字信息；返回空则不输出易错字行
7. 将题目和解析格式化为清晰的文本，直接在对话框中输出
8. 输出格式：先输出题目，然后用【答案与解析】标题分隔，输出每题的答案和易错字提示（按需标注）
"""
```

| 4 | 实现Agent核心流程（意图解析含题型/数量 + 工具编排 + 对话框输出） | 步骤3 |
| 5 | 对接前端对话界面 | 步骤4 |
| 6 | 端到端测试与优化 | 全部 |

### 6.2 关键技术风险

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 教师无法提供结构化的考试提纲数据 | 知识库无法构建 | 协助教师整理，或从公开渠道获取浙江省中考提纲 |
| 考试频率数据缺失或不准确 | 出题质量下降 | 频率数据为**可选**，缺失时退化为均匀随机；先用教师经验估算，后续根据使用反馈迭代 |
| 理解性默写题干泄露答案或不规范 | 题目质量与教学严谨性下降 | LLM 生成题干时校验不得包含答案原句；必要时结合 `comprehension_hints` 辅助生成 |
| LLM生成的题目与原文不一致 | 准确性问题 | 通过知识库约束出题范围，LLM只负责格式化和解析，不负责"背诵" |
