# 技术选型清单

本文件列出实现三个核心功能所需的全部技术能力及已确定的选型方案。

---

## 一、大语言模型（LLM）

| 能力 | 说明 | 涉及功能 |
|------|------|----------|
| 古诗文背诵与出题 | 准确回忆课本古诗文原文，生成上下句填空题 | 功能1 |
| 文学知识问答 | 回答作者生平、创作背景、文学流派等问题 | 功能2 |
| 知识图谱生成 | 根据课文/作者/主题生成结构化关联数据（节点与关系） | 功能2 |
| 作文题目生成 | 根据主题、难度、文体要求生成中考或普通作文题 | 功能3 |
| 多轮对话 | 支持教师与系统反复协商，迭代优化输出 | 功能3（出题） |

**已确定方案**：
- **DeepSeek API**：中文能力强，成本低，API 兼容 OpenAI 格式。

> 备选（暂不采用）：通义千问 API、Kimi API、本地部署开源模型。

---

## 二、后端服务

**已确定方案**：Python 生态。

| 能力 | 说明 | 选型方案 |
|------|------|----------|
| HTTP API 服务 | 提供 RESTful API 供前端调用 | FastAPI |
| LLM 调用封装 | 封装 DeepSeek 的调用逻辑 | `openai` Python SDK（DeepSeek 兼容 OpenAI 格式） |
| 对话状态管理 | 保存多轮对话上下文，支持出题协商 | Redis |
| 配置管理 | API Key、模型参数等敏感信息管理 | `python-dotenv`、`pydantic-settings` |

---

## 三、前端界面

**已确定方案**：Vue 3 + Ant Design Vue。

| 能力 | 说明 | 选型方案 |
|------|------|----------|
| 对话式 UI | 类似 ChatGPT 的聊天界面，支持文本输入输出 | Vue 3 + Ant Design Vue 组件 |
| 表单与筛选器 | 功能1的年级/学期/体裁/重要性选择控件 | Ant Design Vue 表单组件 |
| 可视化图谱 | 功能2的力导向图展示 | ECharts Graph（兼容 Vue 3）或 D3.js |
| Markdown 渲染 | 展示模型输出的 Markdown 格式内容 | `marked.js` 或 `vue-markdown-render` |
| 代码/文本复制 | 一键复制题目、答案等内容到剪贴板 | 浏览器原生 Clipboard API |
| 响应式布局 | 适配电脑、平板等不同屏幕尺寸 | Ant Design Vue 栅格系统 + CSS Flexbox |

> 选择 Vue 3 + Ant Design Vue 的理由：组件丰富成熟，文档完善，上手快，适合快速搭建管理后台/对话型 Web 应用。

---

## 四、数据存储

**已确定方案**：MySQL。

| 能力 | 说明 | 选型方案 |
|------|------|----------|
| 用户对话记录 | 保存教师与系统的历史对话 | MySQL |
| 本地知识库 | 存储初中语文课文清单、作者信息、标准古诗文文本 | MySQL（结构化数据）或 JSON 文件（初期轻量） |
| 考卷数据导入 | 教师导入的历史考卷及出题频率统计 | CSV/Excel 解析（`pandas`、`openpyxl`）后写入 MySQL |
| 缓存 | 缓存 LLM 调用结果，减少重复请求和费用 | 本地文件缓存（开发期）、后续可引入 Redis |

> 关于云服务器：MySQL 后期完全可以部署在云服务器上（如阿里云、腾讯云），前端和后端也可以统一部署到云端，供教师随时访问。目前先在本地运行测试。

---

## 五、部署与运维

**已确定方案**：本地开发环境运行测试。

| 能力 | 说明 | 选型方案 |
|------|------|----------|
| 本地开发环境 | 开发者在本地运行测试 | Python 虚拟环境（`venv`）+ `uvicorn` 本地服务器 + Vue 3 开发服务器 |
| 生产部署 | 将应用部署到服务器供教师访问 | **暂不实现**，后续视情况选择云服务器 / Docker / Serverless |
| 环境变量管理 | 管理 API Key、数据库连接串等敏感配置 | `.env` 文件 |
