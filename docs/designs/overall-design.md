## 二、技术选型确认

| 层级 | 选型 | 说明 |
|------|------|------|
| LLM | DeepSeek API | 中文古诗文与作文任务表现优异，成本可控，兼容 OpenAI 格式 |
| 后端 | Python + FastAPI | 异步高性能，现代 Python Web 框架，自动生成 OpenAPI 文档 |
| 前端 | Vue 3 + Ant Design Vue | 组件丰富，文档完善，适合管理后台/对话型 Web 应用 |
| 数据库 | MySQL | 结构化存储对话记录、配置、后续知识库 |
| 缓存 | Redis | 会话状态、对话上下文缓存 |
| 部署 | 本地开发环境 (uvicorn) | 当前阶段本地运行，后续迁移云服务器 |

## 三、功能模块设计

### 3.1 功能1：古诗文默写出题

**核心流程**：
1. 教师在前端配置筛选条件（年级、学期、体裁、重要性）。
2. 前端将条件发送至后端 `/api/v1/quiz/generate`。
3. 后端组装 Prompt，调用 DeepSeek API 生成题目。
4. 后端解析 LLM 输出为结构化 JSON，返回给前端。
5. 前端展示题目列表与答案列表，支持一键复制。

**Prompt 策略**：
```
角色：你是一位资深初中语文教师。
任务：根据以下筛选条件，生成 {数量} 道古诗文上下句填空题。
条件：
- 年级：{grade}
- 学期：{semester}
- 体裁：{genre}
- 重要性：{importance}
- 范围：浙江省部编版初中语文教材

要求：
- 每题给出上句或下句，留空让填写另一句。
- 每题标注出处（课文名）。
- 以 JSON 格式输出，字段：question, answer, source。
- 确保原文准确，不自行编造。
```

**API 设计**：
```http
POST /api/v1/quiz/generate
Body: { grade, semester, genre, importance, count }
Response: { questions: [{ question, answer, source }] }
```

### 3.2 功能2：课文背景与知识图谱

**核心流程**：
1. 教师输入自然语言指令（如"查看《岳阳楼记》的作者生平"）。
2. 前端将指令发送至后端 `/api/v1/knowledge/query`。
3. 后端识别指令类型（作者生平 / 创作背景 / 文学流派 / 知识图谱），组装对应 Prompt。
4. 后端调用 DeepSeek API 获取结构化内容。
5. 若指令为"知识图谱"，额外调用一次 API 生成图谱数据（节点与关系），前端使用 ECharts Graph 渲染力导向图。

**Prompt 策略（文字内容）**：
```
角色：你是一位初中语文教学专家。
任务：关于课文《{title}》，请提供以下信息：
- 作者生平（姓名、字号、朝代、主要经历）
- 创作背景（创作时间、动机、历史社会环境）
- 文学流派（所属流派、流派特点、代表人物）

以结构化 Markdown 输出，方便教师复制到教案。
```

**Prompt 策略（知识图谱）**：
```
角色：你是一位文学知识图谱专家。
任务：以《{title}》为中心节点，生成一个知识图谱。
要求：
- 输出 JSON 格式，包含 nodes（节点数组）和 edges（关系数组）。
- 节点至少包含：课文本身、作者、朝代、同作者其他作品。
- 边标注关系类型："作者""朝代""同作者作品"等。
- 节点和边数量适中，适合在课堂投屏展示（建议 5-15 个节点）。
```

**API 设计**：
```http
POST /api/v1/knowledge/query
Body: { instruction: "查看《岳阳楼记》的作者生平" }
Response: { type: "text", content: "..." }

POST /api/v1/knowledge/graph
Body: { title: "岳阳楼记", custom_relation?: "唐宋八大家" }
Response: { type: "graph", nodes: [...], edges: [...] }
```

### 3.3 功能3：命题作文出题

**核心流程**：
1. 教师输入主题和题目类型（中考题/普通题），前端发送给后端。
2. 后端创建会话（存入 Redis/MySQL），组装 Prompt 生成初稿。
3. 教师提出修改意见，前端发送会话 ID + 意见给后端。
4. 后端读取历史对话上下文，继续对话，返回优化后的题目。
5. 教师确认最终版本，后端标记会话完成，输出最终题目。

**会话管理**：
- 使用 Redis 缓存活跃会话（最近 30 分钟有交互）。
- 会话过期时间：2 小时。
- 会话数据格式：`{ session_id, messages: [{ role, content }], config: { theme, type, style, word_count, difficulty, extra }, status }`

**Prompt 策略**：
```
角色：你是一位资深中考语文命题专家。
任务：为初中语文教师生成作文题目。

当前配置：
- 主题：{theme}
- 类型：{type}（中考题 / 普通题）
- 文体：{style}
- 字数：{word_count}
- 难度：{difficulty}
- 附加要求：{extra}

要求：
- 若为中考题：提供一段材料（诗歌/名言/故事），给出 2-3 个话题方向，要求题目自拟，字数达标。
- 若为普通题：简洁给出主题和写作要求。
- 仅输出题目本身，不输出写作提示、评分标准、范文。
```

**API 设计**：
```http
POST /api/v1/essay/start
Body: { theme, type, style, word_count, difficulty, extra }
Response: { session_id, draft: "..." }

POST /api/v1/essay/iterate
Body: { session_id, feedback: "换个材料" }
Response: { draft: "...", iteration_count }

POST /api/v1/essay/finalize
Body: { session_id }
Response: { final_title: "..." }
```

## 四、数据模型

### 4.1 对话记录 (conversations)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 自增 |
| session_id | VARCHAR(64) | 会话唯一标识 |
| feature | VARCHAR(32) | 所属功能：quiz / knowledge / essay |
| messages | JSON | 消息数组 [{ role, content, timestamp }] |
| config | JSON | 功能配置参数 |
| status | VARCHAR(16) | active / completed / expired |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 4.2 用户配置 (user_settings) —— 预留
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 自增 |
| default_grade | VARCHAR(16) | 默认年级 |
| default_difficulty | VARCHAR(16) | 默认难度 |
| templates | JSON | 作文出题模板（后续迭代） |

## 五、前端页面设计

### 5.1 页面结构

```
┌────────────────────────────────────────┐
│ 顶部导航栏：初中语文教师智能体            │
├────────────┬───────────────────────────┤
│            │                           │
│  左侧菜单  │      主内容区              │
│  - 对话首页 │      (动态切换)            │

│  - 作文出题 │                           │

```

### 5.2 各功能页面

| 功能 | 页面元素 | 说明 |
|------|---------|------|
| 对话首页 | 聊天输入框、历史会话列表 | 通用入口，可触发任意功能 |
| 默写出题 | 筛选表单(年级/学期/体裁/重要性)、题目展示区、答案区、复制按钮 | 一次生成，无需多轮对话 |
| 知识图谱 | 指令输入框、文字内容区、ECharts 图谱区、全屏按钮 | 支持文字和可视化双输出 |
| 作文出题 | 对话式聊天界面、配置面板(侧滑或折叠)、确认/重新生成按钮 | 多轮协商模式 |

### 5.3 通用组件

- `ChatInput`：带发送按钮的文本输入框，支持 Markdown 渲染。
- `CopyButton`：一键复制文本到剪贴板。
- `FullscreenToggle`：用于知识图谱全屏展示。
- `MarkdownRenderer`：安全渲染 LLM 输出的 Markdown 内容。

## 六、错误处理与降级策略

| 场景 | 策略 |
|------|------|
| LLM API 超时/失败 | 返回友好错误提示，建议用户重试；记录日志 |
| LLM 输出格式不符合预期 | 前端做容错解析，无法解析时展示原始文本 |
| Redis 不可用 | 降级为内存会话存储（丢失重启后数据，但不阻塞使用） |
| 模型幻觉（古文错字） | 界面展示免责声明；提供"反馈错误"按钮收集数据 |

## 七、安全与隐私

- API Key 存储在 `.env` 文件，不提交到 Git。
- 前端不直接调用 DeepSeek API，所有 LLM 调用走后端代理。
- 当前 MVP 不实现用户认证，但会话数据按 `session_id` 隔离。
