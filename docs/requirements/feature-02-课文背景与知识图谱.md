# 功能2：课文背景与知识图谱

## 一、功能概述

为初中语文教师提供单篇课文的深度拓展信息，并以可视化知识图谱的形式呈现关联内容，辅助教师备课与课堂展示。

## 二、触发方式

- **主动技能触发**：教师需明确发出指令方可唤起本功能。
- **支持的自然语言指令示例**：
  - "查看《岳阳楼记》的作者生平信息"
  - "查看《岳阳楼记》的创作背景"
  - "查看《岳阳楼记》的文学流派"
  - "查看《岳阳楼记》的知识图谱"
- **不自动关联**：本功能不与其他功能（如功能1默写出题）自动联动触发，保持独立性。

## 三、拓展内容范围（第一轮）

第一轮先实现以下三个核心维度，后续根据教师反馈逐步扩展：

| 维度 | 输出内容 |
|------|----------|
| 作者生平 | 姓名、字号、朝代、主要经历 |
| 创作背景 | 创作时间、创作动机、当时的历史社会环境 |
| 文学流派 | 所属流派（如唐宋八大家、豪放派/婉约派）、流派特点、代表人物 |

> **后续待扩展维度**：课文主旨与艺术特色、后人评价/文学史地位、重点字词解析、与中考的关联等。

## 四、知识图谱设计

### 4.1 图谱形态

- **采用形态 B：可视化关系网络图**
  - 中心节点为当前课文（如《岳阳楼记》）。
  - 周围节点为关联实体（作者、朝代、关联人物、关联作品、主题等）。
  - 节点之间用连线连接，并标注关系类型（如"作者""同时代""同主题""同属流派"）。
  - 采用**力导向图（Force-Directed Graph）**布局，节点可自由分布，支持拖拽、缩放交互。

### 4.2 默认关联规则

- **默认关联方向：同作者**
  - 以当前课文为中心，向外辐射关联该作者的其他作品。
  - 示例：查询《岳阳楼记》时，图谱中心为《岳阳楼记》，周围节点包含范仲淹及其其他作品（如《渔家傲·秋思》）。

### 4.3 自定义关联规则

- 教师可现场提出自定义关联关系，系统动态生成对应的知识图谱。
- **示例**：
  - 教师指令："查看唐宋八大家"
  - 系统以"唐宋八大家"这一关系为中心，生成包含韩愈、柳宗元、欧阳修、苏洵、苏轼、苏辙、王安石、曾巩等人的知识图谱。
  - 教师指令："查看同时代的其他作家"
  - 系统以当前作者为中心，关联同时代的其他重要作家及其代表作。

## 五、输出形式

| 用途 | 说明 |
|------|------|
| 辅助备课 | 直接在 Web 页面阅读，图文并茂 |
| 复制到教案/PPT | 支持一键复制文本内容，格式清晰 |
| 课堂展示（投屏） | 可视化图谱支持全屏展示，适合投屏给学生观看；文字内容排版清晰，适合投影 |

## 六、交互流程示例

```
教师输入：查看《岳阳楼记》的知识图谱

系统输出：
1. 文字区：
   - 作者生平：范仲淹（989-1052），字希文，北宋政治家、文学家...
   - 创作背景：庆历六年，范仲淹因庆历新政失败被贬邓州...
   - 文学流派：不属于唐宋八大家，但与欧阳修等共同推动宋代古文运动...

2. 图谱区（可视化）：
   [中心节点：岳阳楼记]
      ├── [作者] 范仲淹
      │      └── [同作者作品] 渔家傲·秋思
      ├── [朝代] 北宋
      │      └── [同时代] 欧阳修 ── [作品] 醉翁亭记
      └── [主题] 忧国忧民
             └── [同主题] 杜甫 ── [作品] 春望
```

## 七、技术提示

- 可视化图谱可选用前端库如 **D3.js**、**ECharts Graph** 或 **Cytoscape.js** 实现。
- 图谱数据由大模型（DeepSeek）根据课文实时生成，前期不引入固定知识库，允许教师在发现错误时反馈修正。

## 八、最小可行版本（MVP）

第一轮实现以下功能：

1. 教师输入指令触发本功能。
2. 系统输出该课文的作者生平、创作背景、文学流派三段文字内容。
3. 系统默认生成"同作者"关联的可视化知识图谱。
4. 支持教师输入自定义关联关系，动态生成新的知识图谱。
5. 文字内容支持一键复制；图谱支持全屏展示。

## 九、后续迭代方向
